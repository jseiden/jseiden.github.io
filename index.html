Ternary Operator

The ternary operator is a useful shortcut for Javascript conditionals (aka if statements). The ternary can be thought of like an if...else statement; if x is true, perform action y. Otherwise, perform action z. Consider the following if...else statement:

if( animal === "dog" ) {
  console.log( "bark" );
} else {
  console.log( "meow" );
}

Now consider the equivalent ternary: 

animal === "dog" ? console.log( "bark" ) **:  console.log( "meow" );

The word ternary describes something consisting of three parts so it should make sense that this construct has three parts. The borders are indicated by a question mark(?) and a colon(:). if x is true ? perform action y : otherwise, perform action z

  1. The leftmost section is the condition that must be met. It is equivalent to the code inside the initial parens of an if..else statement.

  2. The middle section is the action that will be performed if the left section evaluates to true, equivalent to the code inside the first pair of brackets of an if...else statement.

  3. The right section is the action that will be performed if the condition is false. This is like the code within the brackets after the word else in an if..else statement.

The following is an even shorter, more elegant way to code a basic if...else:

console.log( animal == "dog" ? "bark" : "meow" );

It is definitely true that some programmers consider the standard if statement more readable, despite its verbosity. Just like standard if...else statements, ternary conditional logic can get a lot more complex than the examples I've given you. However, using ternary operators of any complexity is a valuable practice that can save you both space and time.

The Pros and Cons of Yeoman

Yeoman is a very powerful scaffolding tool. I've only recently learned to appreciate the practice of scaffolding. Starting a project with a preset layout can allow a programmer to bypass a lot of setup and commence coding immediately. It is very satisfying to start sculpting the innards of your project 5 minutes after sitting down. If you are confident that your chosen template is legit, this can also be a way to reinforce best practices and become comfortable with common file structures that you wouldn't have made yourself.

First install yo, Yeoman's command line tool.

$ npm install -g yo

Now install a scaffold, which is referred to as a generator. For this example we will use an Angular generator. There are multiple Angular generators, each with a different stack or setup, but we will go with the official one. 

$ npm install generator-angular

Once Yeoman and the Angular generator are installed with npm, the following code is all it takes to get started:

$ mkdir myApp
$ cd myApp
$ yo angular

Yo will then guide you through some basic options such as including Twitter Bootstrap or Sass. Pretty easy, right? One must also consider the pitfall's of such a tool. Yeoman is opinionated. It also abstracts some crucial steps in the setup. For now, I'm not using Yeoman on personal projects. It creates some valuable shortcuts that I intend to use in the future, but in the meantime, the details are important. In the end, it's worth considering both the pros and cons of before start your next project.

Types of Loops and When to Use Them

Loops are an essential programming concept. They do an extreme amount of work for you. Consider the following code.

console.log(1);
console.log(2);
console.log(3);
console.log(4);
console.log(5);
console.log(6);
console.log(7);
console.log(8);
console.log(9);
console.log(10);

This is only ten consecutive console logs and already a somewhat tedious. Now take a look at the for loop equivalent. FYI, if you don't recognize the ++ operator, all it does is increment the variable it's placed after. So i++ is equivalent to i = i + 1;

for(var i =  1; i < 11; i++){
  console.log(i)
}

With only three lines of code we can achieve the same result without once repeating ourselves. Things become even cooler when we change 11 to 101 or even 1001.

While loops are also worth knowing. 

var i = 0; 
while(i < 11){
  console.log(i)
  i++;
}

This example while loop does the same thing as the example for loop. A for loop is most useful when you know the number of repetitions you want. However, if you don't know that number or it could change, you'll probably want to use a while loop. This is the end of my post for now. Even if you're a beginner, don't be afraid to use loops in your code. They promote cleaner, less repetitive code, and make you a more efficient programmer.

Array-like Objects in Javascript

More than once have I turned to Google after an Javascript Array method returned something unexpected. This is most often due to an Array-like Object. The standard example is the arguments object which contains all the args passed to a certain function invocation. This object has the familiar brackets seen on an array and contains a comma-separated list of primitives and/or objects. This sounds like an array, right? Nope. Let the following code illustrate this confusing situation:

// here you can see a regular old array 
var stuff = [1, 3, 5];
console.log(stuff.length);
console.log(stuff.pop());
// 3
// 5
var fn = function(a, b, c){
  console.log(arguments);
  console.log(arguments.length);
  console.log(arguments.pop());
}

fn(1, 3, 5);
// arguments: [1, 3, 5]
// TypeError: arguments.pop is not a function

Despite the arguments object's uncanny resemblance to a real array, it doesn't have a pop method, nor does it have many of the other useful native methods of the Array object. You can see that it does possess a length property which, admittedly, is kind of nice but the full functionality of the Array object is within reach if you can remember a trick or two.

The vanilla.js way to get around this is as follows: 

var fn = function(a, b, c){
  var argsToArr = Array.prototype.slice.call(arguments);
  console.log(arguments);
  console.log(argsToArr.pop());
}
fn(1, 2, 3);
// [1, 3, 5];
//  3

One can also use the underscore.js toArray method to achieve the same results. 

var fn = function(a, b, c){
  console.log(_.toArray(arguments).pop());
}
fn(1, 2, 3);
// 3

If you already knew this, thanks for reading. If you didn't, go forth and code with one more tool in your belt.

Basic Node and Express Servers

When learning a new technology, I find minimal examples the most helpful. Express is a server-side Javascript framework built on Node.js that streamlines and abstracts the server-building process. Here are two bare-bones servers for comparison.

Node server

var http = require("http");
var port = 3000;
var server = http.createServer(function(request, response){
  response.writeHead(200, {"Content-type": "text/plain"});
  response.end("This is a basic node server!");
});
server.listen(port);

Express server

var express = require("express");
var app = express();
var port = 3000;
app.get('/', function(request, response){
  response.send("This is a basic express server");
}
app.listen(port);

What is a server meant to do? A server, at its most basic, is a computer or a computer program that takes in requests and sends back a response. In this case, the request is an http request and the response is the file(s) that make up a web page or part of it.

These servers are nearly identical in length but are slightly different. What is the point, you might ask, in writing a blog post that describes two very similar sections of code? I think it's important to compare before contrasting. The subtle differences in the code foreshadow much more salient features of Express.

Note that the basic Express server doesn't import the http module that the node version does. This is one of node's built in modules that Express doesn't explicitly require, As you build more servers, you'll notice more subtle abstractions that differentiate it from plain Node. The real power of the Express framework is in the middleware, which I intend to define and discuss in an upcoming post.
