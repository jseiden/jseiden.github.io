The Pros and Cons of Yeoman.

Yeoman is a very powerful scaffolding tool. I've only recently learned to appreciate the practice of scaffolding. Starting a project with a preset layout can allow a programmer to bypass a lot of setup and commence coding immediately. It is very satisfying to start sculpting the innards of your project 5 minutes after sitting down. If you are confident that your chosen generator is legit, this can also be a way to reinforce best practices and become comfortable with common file structures that you wouldn't have made yourself. Once yeoman and the angular generator are installed with npm, the following code is all it takes to get started:

$ mkdir myApp
$ cd myApp
$ yo angular

Yeoman's command line tool will then guide you through some basic options such as including Twitter Bootstrap or the Sass. Pretty easy, right? One must also consider the pitfall's of such a tool. Yeoman is, by nature, opinionated. It also abstracts some crucial steps in the setup. For now, I'm not using Yeoman. It creates some valuable shortcuts that I intend to use in the future, but in the meantime, the details are important. In the end, it's worth considering both the pros and cons of before start your next project.

Types of Loops and When to Use Them

Loops are an essential programming concept. They do an extreme amount of work for you. Consider the following code.

console.log(1);
console.log(2);
console.log(3);
console.log(4);
console.log(5);
console.log(6);
console.log(7);
console.log(8);
console.log(9);
console.log(10);

This is only ten consecutive console logs and already a somewhat tedious. Now take a look at the for loop equivalent. FYI, if you don't recognize the ++ operator, all it does is increment the variable it's placed after. So i++ is equivalent to i = i + 1;

for(var i =  1; i < 11; i++){
  console.log(i)
}

With only three lines of code we can achieve the same result without once repeating ourselves. It becomes even cooler when we change 11 to 101 or even 1001. It is possible to increment this number until the computer can't handle it. While loops are also worth knowing. 

var i = 0; 
while(i < 11){
  console.log(i)
  i++;
 }

This example while loop does the same thing as the example for loop. A for loop is most useful when you know the number of repetitions you want. However, if you don't know that number or it could change, you'll probably want to use a while loop. This is the end of my post for now. Even if you're a beginner, don't be afraid to use loops in your code. They promote cleaner, DRYer code, and make you a more efficient programmer.

Array-like Objects in Javascript

More than once have I turned to Google after an Javascript Array method returned something unexpected. This is most often due to an Array-like Object. The standard example is the arguments object which contains all the args passed to a certain function invocation. This object has the familiar brackets seen on an array and contains a comma-separated list of primitives and/or objects. This sounds like an array, right? Nope. Let the following code illustrate this confusing situation:

// here you can see a regular old array 
var stuff = [1, 3, 5];
console.log(stuff.length);
console.log(stuff.pop());
// 3
// 5
var fn = function(a, b, c){
  console.log(arguments);
  console.log(arguments.length);
  console.log(arguments.pop());
}

fn(1, 3, 5);
// arguments: [1, 3, 5]
// TypeError: arguments.pop is not a function

Despite the arguments object's uncanny resemblance to a real array, it doesn't have a pop method, nor does it have many of the other useful native methods of the Array object. You can see that it does possess a length property which, admittedly, is kind of nice but the full functionality of the Array object is within reach if you can remember a trick or two.

The vanilla.js way to get around this is as follows: 

var fn = function(a, b, c){
  var argsToArr = Array.prototype.slice.call(arguments);
  console.log(arguments);
  console.log(argsToArr.pop());
}
fn(1, 2, 3);
// [1, 3, 5];
//  3

One can also use the underscore.js toArray method to achieve the same results. 

var fn = function(a, b, c){
  console.log(_.toArray(arguments).pop());
}
fn(1, 2, 3);
// 3

If you already knew this, thanks for reading. If you didn't, go forth and code with one more tool in your belt.

Basic Node and Express Servers

When learning a new technology, I find minimal examples the most helpful. Express is a server-side Javascript framework built on Node.js that streamlines and abstracts the server-building process. I have built two bare-bones servers for comparison.

a basic node server

var http = require("http");
var port = 3000;
var server = http.createServer(function(request, response){
  response.writeHead(200, {"Content-type": "text/plain"});
  response.end("This is a basic node server!");
});
server.listen(port);

a basic Express server

var express = require("express");
var app = express();
var port = 3000;
app.get('/', function(request, response){
  response.send("This is a basic express server");
}
app.listen(port);

What is a server meant to do? A server, at its most basic, is a computer or a computer program that takes in requests and sends back a response. In this case, the request is an http request and the response is the file(s) that make up a web page or part of it.

These servers are nearly identical in length but are slightly different. What is the point, you might ask, in writing a blog post that describes two very similar sections of code? I think it's important to compare before contrasting. The subtle differences in the code foreshadow much more salient features of Express.

Note that the basic Express server doesn't import the http module that the node version does. This is one of node's built in modules that Express doesn't explicitly require, As you build more servers, you'll notice many more subtle abstractions that differentiate it from plain node. The real power of the Express framework is in the middleware, which I intend to define and discuss in an upcoming post.
